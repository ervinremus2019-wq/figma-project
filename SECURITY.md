# Security Policy

## Reporting a Vulnerability
If you discover a security vulnerability, **do not open an issue**. Instead, send an email to [ervinremus2019@gmail.com](mailto:ervinremus2019@gmail.com) with the following details:
- Description of the vulnerability
- Steps to reproduce
- Your name and contact information (optional)

We will acknowledge your email within 48 hours and provide a timeline for a fix.

## Supported Versions
Only the latest version of this project is currently supported with security updates.

## Security Best Practices
- Always use environment variables for sensitive data (e.g., API keys).
- Regularly update dependencies to patch known vulnerabilities.
- Use HTTPS for all external requests.
- Never commit secrets or sensitive information to the repository.
- Use `.gitignore` to exclude sensitive files.

## Dependencies
This project uses third-party dependencies. Always review their security advisories and update regularly.
